# KPTA 회고 프레임워크

저널의 회고 양식은 KPTA(Keep-Problem-Try-Action) 프레임워크를 따릅니다.
KPT 회고에 실행 계획(Action)을 추가하여, 회고가 구체적인 행동 변화로 이어지도록 합니다.

## 4가지 관점

### Keep (유지할 것)
- 잘 작동하고 있어서 **계속 유지**해야 할 practice, 습관, 도구
- 이번 작업에서 효과적이었던 접근 방식
- 예: "TDD로 먼저 테스트를 작성한 덕분에 리팩토링 시 안전망이 되었다"

### Problem (문제점)
- 작업 중 겪은 **어려움, 비효율, 실수**
- 반복되는 패턴이라면 특히 중요하게 기록
- 원인 분석을 함께 기록 (단순 현상 나열이 아닌 **왜** 발생했는지)
- 예: "캐시 무효화 타이밍을 잘못 잡아 30분간 디버깅 — 캐시 레이어 문서화 부재가 원인"

### Try (시도할 것)
- Problem을 해결하기 위해 **다음에 시도해볼** 아이디어나 방법
- Keep을 더 강화할 수 있는 개선안도 포함
- 예: "캐시 레이어에 TTL 설정 가이드 문서를 작성한다"
- **세션 데이터에서 명확히 도출되는 항목만 작성**
- 쓸 내용이 없으면 비워두고 사용자가 직접 채우도록 함
- 과거 저널의 Problem 패턴을 바탕으로 **제안/추천** 형태로 작성 (단정 금지)

### Action (실행 계획)
- Try 항목 중 **즉시 실행할 구체적인 행동**으로 전환
- 예: "이번 주 금요일까지 `docs/cache-guide.md` 작성 완료"
- **세션 데이터에서 명확히 도출되는 항목만 작성**
- 쓸 내용이 없으면 비워두고 사용자가 직접 채우도록 함
- 과거 저널의 Try/Action 이력을 바탕으로 **제안/추천** 형태로 작성 (단정 금지)

## 작성 기준

### Try / Action 작성 원칙
- **상상하지 않는다**: 세션 데이터에서 근거를 찾을 수 없으면 비워둔다
- **제안한다**: 과거 저널의 반복 패턴이나 미완료 Action을 바탕으로 추천하되, 확정이 아닌 제안으로 작성
- **사용자가 완성한다**: Try와 Action은 사용자가 최종 판단하여 채우는 영역

### 분량 기준
- 일별 저널: 각 관점당 1~3개 항목
- 주별 저널: 각 관점당 3~5개 항목 (일별 저널에서 패턴 추출)
- 월별 저널: 각 관점당 3~5개 항목 (주별 저널에서 트렌드 추출)

### 작성 톤
- 자기 성찰적이되 지나치게 감성적이지 않게
- 사실 기반으로 작성 (세션 데이터에서 근거를 찾을 수 있어야 함)
- 다음 행동으로 이어지는 실용적인 관점 유지

## 저널 내 적용 위치

저널 템플릿에서 회고 섹션은 KPTA 구조를 따릅니다:

```markdown
## 회고

### Keep
- ...

### Problem
- ...

### Try
- ...

### Action
- ...
```

## 주별/월별 저널에서의 활용

- **주별**: 일별 저널의 KPTA를 취합하여 반복 패턴 식별
- **월별**: 주별 KPTA를 종합하여 장기 트렌드와 성장 포인트 도출
- Action 항목의 완료 여부를 추적하여 실행력 점검
